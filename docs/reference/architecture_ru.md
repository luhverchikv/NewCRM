1|# Архитектура
2|
3|## Слои (текстовая диаграмма)
4|Обновление -> Промежуточное ПО -> Обработчики -> Службы -> UnitOfWork -> Репозитории -> База данных
5|
6|## Узоры
7|Репозиторий хранит запросы рядом с моделями.
8|```питон
9|класс UserRepository(AbstractRepository[UserOrm, int]):
10|модель = ПользовательОрм
11|```
12|Единица работы охватывает одну транзакцию для каждого запроса.
13|```питон
14|асинхронно с self._uow:
15|пользователь = ожидайте self._uow.users.get(user_id)
16|```
17|Службы и утилиты внедрения зависимостей.
18|```питон
19|@aiogram_inject
20|async def start (сообщение: Сообщение, user_service: FromDishka[UserService]) -> Нет:
21|ожидайте user_service.register_user(message.from_user.id)
22|```
23|
24|## Поток данных
25|`обновление -> маршрутизатор -> обработчик -> сервис -> uow -> репозиторий -> база данных -> обработчик -> ответ`
26|
27|## Обоснование структуры каталогов
28|- `source/telegram/` содержит ввод/вывод пограничного уровня (обработчики, фильтры, клавиатуры).
29|- `source/services/` сохраняет бизнес-логику и оркестровку.
30|- `source/database/` содержит модели, репозитории и UoW.
31|- `source/factory/` связывает бота, диспетчера, приложение и контейнер DI.
